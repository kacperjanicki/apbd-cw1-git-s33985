1. - Fast forward wykona się wtedy, gdy gałąź do której są scalane zmiany nie przesunęła się do przodu od utworzenia nowej gałęzi
   - Merge commit powstanie gdy historia obu gałęzi się rozeszła, obie gałęzie mają nowe unikalne commity
2. - Merge zachowuje prawdziwą historię pracy. Widać kiedy gałąź została stworzona i połączona.
   - Rebase przepisuje historię, aby była liniowa. Graf jest w postaci lini prostej, nie ma takiego widocznego "oczka" jak przy merge
